    1.2   Topic 结构

    Topic Title Example

    这是 topic 第一段。

    这是 topic 的第二段。

    topic 结构一般用在文章开头，或某段开头，或是 transition 开始时，用于点明主题。上面这个 topic 的例子的原文为：

    .. topic:: Topic Title Example

    Lists:

- item 1. 第一项前和最后一项后必须各留出一空行.
- item 2. List 由 "*", "+", "-" 等符号引导。

- item 3. 中间各项之间可留空行也可省略.

  1. nested item 1. 此前又须留出一空行。数字型的list接
     受这几种格式：

     a. 阿拉伯数字 1, 2, 3 ...
     #. 大写或小写字母 A, B, ..., Z. a, b, ..., z.
     #. 大写或小写罗马数字 I, II, ... 或 i, ii, ...

    Literal Blocks:

::

  Whitespace, newlines, blank lines, and all kinds of
  markup is preserved by literal blocks.

Per-line quoting can also be used on unindented
literal blocks (顶格写时每行的 ``>`` 符号不可少)::

> Useful for quotes from email and
> for Haskell literate programming.

Block Quotes:

Block quotes 格式为相对前一行进行缩进，

    Block quotes 仅仅是缩进，并不会像 literal blocks
    或是 line blocks 那样完全照搬格式。例如这一段在渲
    染后的排版断行不会完全和原文一模一样。

        Nested block quotes.
